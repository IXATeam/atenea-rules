■ SINTAXIS C# OBLIGATORIA:
- NUNCA uses LINQ (.Where, .Cast, .Select, .First, .ToList, .Any)
- NUNCA uses interpolacion de strings ($"texto {var}") — usa concatenacion: "texto " + var
- NUNCA uses lambdas (=>) excepto en Sort((a,b) => ...)
- USA foreach en lugar de LINQ para iterar colecciones
- Las variables 'document', 'uidoc' ya estan disponibles — NO las redeclares
- ElementId requiere sufijo L: new ElementId(12345L)
- Para obtener el valor numerico de ElementId usa: id.Value (no .IntegerValue)
- SubTransaction: usala para modificaciones dentro de una Transaction del host

■ REGLA 1 — COORDENADAS Y UNIDADES:
Revit usa pies internamente. Convierte siempre: metros * 3.28084 = pies
Ejemplo: 4 metros = 4.0 * 3.28084 = 13.12336 pies
Para convertir de pies a metros: pies * 0.3048
XYZ siempre en pies: new XYZ(x_pies, y_pies, z_pies)

■ REGLA 2 — OBTENER NIVELES:
Level nivel = null;
FilteredElementCollector col = new FilteredElementCollector(document);
col.OfClass(typeof(Level));
foreach (Level l in col) {
    if (l.Name == "Nivel 1") { nivel = l; break; }
}
if (nivel == null) { foreach (Level l in col) { nivel = l; break; } }
PARA OBTENER EL NIVEL DE LA VISTA ACTIVA:
Level nivel = null;
ViewPlan vp = uidoc.ActiveView as ViewPlan;
if (vp != null) nivel = document.GetElement(vp.LevelId) as Level;
if (nivel == null) {
    FilteredElementCollector c2 = new FilteredElementCollector(document);
    c2.OfClass(typeof(Level));
    foreach (Level l in c2) { nivel = l; break; }
}
NUNCA hagas: View as Level — un View NUNCA es un Level.

■ REGLA 3 — REJILLAS (GRID):
Las lineas para Grid.Create() SIEMPRE en el plano XY (Z=0).
Eje vertical (corre en Y): Line.CreateBound(new XYZ(x, -30, 0), new XYZ(x, 30, 0))
Eje horizontal (corre en X): Line.CreateBound(new XYZ(-30, y, 0), new XYZ(30, y, 0))
NUNCA uses Z diferente de 0 para lineas de Grid.
Grid NO tiene propiedad Elevation — NUNCA uses grid.Elevation.
Grid.Create(document, line) — no necesita nivel.
Grid.Curve devuelve un solo Curve, NO un CurveArray.
USO CORRECTO: Curve curva = grid.Curve; XYZ p = curva.GetEndPoint(0);
NUNCA uses grid.Curve.Size ni grid.Curve.get_Item()

■ REGLA 4 — NOMENCLATURA DE INTERSECCIONES DE REJILLA:
Los ejes tienen nombres simples: 'A', 'B', '1', '2' — NO existen ejes 'A1' o 'B2'.
El punto A1 = interseccion del eje 'A' con el eje '1'.
PATRON para obtener punto A1:
Grid ejeA = null, eje1 = null;
FilteredElementCollector cg = new FilteredElementCollector(document);
cg.OfClass(typeof(Grid));
foreach (Grid g in cg) {
    if (g.Name == "A") ejeA = g;
    if (g.Name == "1") eje1 = g;
}
IntersectionResultArray ira;
SetComparisonResult scr = ejeA.Curve.Intersect(eje1.Curve, out ira);
XYZ puntoA1 = (scr != SetComparisonResult.Disjoint && ira != null && ira.Size > 0)
    ? ira.get_Item(0).XYZPoint
    : new XYZ(ejeA.Curve.GetEndPoint(0).X, eje1.Curve.GetEndPoint(0).Y, 0);

■ REGLA 5 — INTERSECCIONES DE CURVAS:
Curve.Intersect() devuelve SetComparisonResult — el array va en parametro OUT.
CORRECTO:
IntersectionResultArray results;
SetComparisonResult scr = c1.Intersect(c2, out results);
if (scr != SetComparisonResult.Disjoint && results != null && results.Size > 0)
    XYZ punto = results.get_Item(0).XYZPoint;
NUNCA: IntersectionResultArray r = c1.Intersect(c2); // ERROR

■ REGLA 6 — MUROS (WALL):
Wall.Create() firma exacta:
Wall.Create(document, curve, wallTypeId, levelId, height, offset, flip, structural)
- height: metros * 3.28084 (ej: 3.0 * 3.28084)
- offset: 0.0
- flip: false
- structural: false
NUNCA uses XYZ.BasisZ como argumento de Wall.Create.
La curva del muro debe estar en Z=0: new XYZ(x, y, 0)
OBTENER TIPO DE MURO (siempre primer disponible):
WallType tipoMuro = null;
FilteredElementCollector cwt = new FilteredElementCollector(document);
cwt.OfClass(typeof(WallType));
foreach (WallType wt in cwt) { tipoMuro = wt; break; }

■ REGLA 7 — PISOS Y LOSAS (FLOOR):
Floor.Create(document, List<CurveLoop>, floorTypeId, levelId)
NO uses el metodo legacy NewFloor().
EJEMPLO:
CurveLoop loop = new CurveLoop();
loop.Append(Line.CreateBound(p1, p2));
loop.Append(Line.CreateBound(p2, p3));
loop.Append(Line.CreateBound(p3, p4));
loop.Append(Line.CreateBound(p4, p1));
List<CurveLoop> loops = new List<CurveLoop>(); loops.Add(loop);
Floor piso = Floor.Create(document, loops, floorTypeId, levelId);
Floor.Create() es solo para LOSAS — requiere FloorType, NO WallType.

■ REGLA 8 — COLUMNAS ESTRUCTURALES Y VIGAS:
COLUMNA: document.Create.NewFamilyInstance(punto, symbol, nivel, StructuralType.Column)
- Primero activa el simbolo: symbol.Activate()
VIGA: document.Create.NewFamilyInstance(lineaEje, symbol, nivel, StructuralType.Beam)
FamilySymbol symbol = null;
FilteredElementCollector cfs = new FilteredElementCollector(document);
cfs.OfClass(typeof(FamilySymbol)).OfCategory(BuiltInCategory.OST_StructuralColumns);
foreach (FamilySymbol fs in cfs) { symbol = fs; break; }
if (symbol != null && !symbol.IsActive) symbol.Activate();

■ REGLA 9 — SELECCIONAR Y LEER ELEMENTOS (PATRON UNIVERSAL):
FilteredElementCollector col = new FilteredElementCollector(document);
col.OfClass(typeof(TipoClase));
foreach (Element e in col) { /* usa e */ }
TIPOS COMUNES:
- Muros: OfClass(typeof(Wall))
- Niveles: OfClass(typeof(Level))
- Rejillas: OfClass(typeof(Grid))
- Pisos: OfClass(typeof(Floor))
- Habitaciones: OfCategory(BuiltInCategory.OST_Rooms)
- Puertas: OfClass(typeof(FamilyInstance)).OfCategory(BuiltInCategory.OST_Doors)
- Ventanas: OfClass(typeof(FamilyInstance)).OfCategory(BuiltInCategory.OST_Windows)
- Columnas: OfClass(typeof(FamilyInstance)).OfCategory(BuiltInCategory.OST_StructuralColumns)
- Vigas: OfClass(typeof(FamilyInstance)).OfCategory(BuiltInCategory.OST_StructuralFraming)
SELECCION ACTIVA:
ICollection<ElementId> sel = uidoc.Selection.GetElementIds();
ESTABLECER SELECCION:
List<ElementId> ids = new List<ElementId>(); ids.Add(elem.Id);
uidoc.Selection.SetElementIds(ids);
NUNCA uses SetElementId() (sin 's') — no existe.

■ REGLA 10 — OBTENER TIPOS DE FAMILIA:
NUNCA filtres por nombre para obtener un tipo — puede no existir.
USA SIEMPRE el primer disponible como fallback:
WallType tipo = null;
FilteredElementCollector c = new FilteredElementCollector(document);
c.OfClass(typeof(WallType));
foreach (WallType wt in c) { tipo = wt; break; }

■ REGLA 22 — CONTORNO DE MUROS (MURO POR SEGMENTOS):
Para crear muros que forman un contorno, usa multiples Wall.Create() — uno por segmento.
NUNCA uses Floor.Create() para crear muros.
double h = 3.0 * 3.28084;
Wall.Create(document, Line.CreateBound(pB2, pC3), tipoMuro.Id, nivel.Id, h, 0, false, false);
Wall.Create(document, Line.CreateBound(pC3, pC5), tipoMuro.Id, nivel.Id, h, 0, false, false);

■ REGLA 23 — ELIMINAR VISTAS (VIEWPLAN, VIEW3D, VIEWSECTION):
NUNCA elimines la vista activa — lanza ArgumentException.
NUNCA elimines vistas que son plantillas (IsTemplate == true).
PATRON CORRECTO:
ElementId vistaActivaId = uidoc.ActiveView.Id;
FilteredElementCollector col = new FilteredElementCollector(document);
col.OfClass(typeof(ViewPlan));
foreach (ViewPlan vp in col) {
    if (vp.Id == vistaActivaId) continue;
    if (vp.IsTemplate) continue;
    try { document.Delete(vp.Id); } catch { }
}
Si necesitas borrar la vista activa, primero activa otra vista:
uidoc.ActiveView = otraVista;
// luego borra la vista anterior

■ REGLA 24 — SECTION BOX EN VISTA 3D:
View3D.SectionBox es de SOLO LECTURA — NUNCA uses vista3d.SectionBox = valor;
El método correcto es SetSectionBox():

BoundingBoxXYZ sectionBox = new BoundingBoxXYZ();
sectionBox.Min = new XYZ(minX, minY, elevInferior);
sectionBox.Max = new XYZ(maxX, maxY, elevSuperior);
vista3d.SetSectionBox(sectionBox);      ← método, NO propiedad
vista3d.IsSectionBoxActive = true;      ← OBLIGATORIO para que aparezca

NUNCA uses: vista3d.SectionBox = sectionBox;  // ERROR: propiedad de solo lectura
SIEMPRE usa: vista3d.SetSectionBox(sectionBox);  // CORRECTO
Sin IsSectionBoxActive = true el Section Box queda invisible aunque esté configurado.
Para desactivarlo: vista3d.IsSectionBoxActive = false;

■ REGLA 25 — SELECCIONAR ELEMENTOS VISIBLES EN VISTA:
ElementIsVisibleFilter NO EXISTE en el API de Revit — NUNCA lo uses.
Para obtener elementos visibles en la vista activa, pasa el viewId
directamente al FilteredElementCollector como segundo argumento:

FilteredElementCollector col = new FilteredElementCollector(document, vista.Id);
col.OfClass(typeof(Wall));
foreach (Wall muro in col) { murosIds.Add(muro.Id); }
uidoc.Selection.SetElementIds(murosIds);

El segundo argumento vista.Id restringe automáticamente a elementos
visibles en esa vista — no se necesita ningún filtro adicional.
NUNCA uses ElementIsVisibleFilter — no existe en el API.

■ REGLA 26 — SELECCION ACTIVA: NUNCA ASUMIR QUE HAY ELEMENTOS SELECCIONADOS:
Cuando el usuario diga "bajo estos muros", "en estos elementos", "a estos objetos",
NUNCA asumir que hay una seleccion activa en Revit.
PATRON CORRECTO: Siempre verificar si hay seleccion, y si no hay, buscar los
elementos por categoria en la vista activa como fallback:

ICollection<ElementId> selIds = uidoc.Selection.GetElementIds();
List<Wall> muros = new List<Wall>();

if (selIds.Count > 0) {
    // Usar seleccion activa
    foreach (ElementId id in selIds) {
        Wall w = document.GetElement(id) as Wall;
        if (w != null) muros.Add(w);
    }
} else {
    // Fallback: tomar todos los muros de la vista activa
    FilteredElementCollector col = new FilteredElementCollector(document, uidoc.ActiveView.Id);
    col.OfClass(typeof(Wall));
    foreach (Wall w in col) muros.Add(w);
}

if (muros.Count == 0) return "Error: No se encontraron muros en la vista activa ni en la seleccion";

■ REGLA 27 — LA SELECCION NO PERSISTE ENTRE EJECUCIONES DE SCRIPTS:
uidoc.Selection.GetElementIds() puede retornar 0 elementos en scripts
subsiguientes aunque el script anterior haya seleccionado elementos.
NUNCA dependas de la seleccion de un script anterior.
PATRON CORRECTO: En scripts que modifican elementos previamente
"seleccionados", busca los elementos directamente por categoria y
filtra por los criterios del usuario:

// En lugar de leer la seleccion, buscar directamente:
FilteredElementCollector col = new FilteredElementCollector(document, uidoc.ActiveView.Id);
col.OfClass(typeof(Wall));
List<Wall> muros = new List<Wall>();
foreach (Wall w in col) muros.Add(w);

// Si el usuario dijo "los muros que seleccionamos", buscarlos en la vista activa.
// La seleccion visual del usuario en Revit es diferente a la seleccion del script.

■ REGLA 28 — Line.CreateBound REQUIERE DOS PUNTOS DISTINTOS:
Line.CreateBound(p1, p2) falla si p1 == p2 (longitud cero).
Antes de crear una linea, verifica que los puntos sean distintos:

XYZ p0 = curva.GetEndPoint(0);
XYZ p1 = curva.GetEndPoint(1);
if (p0.DistanceTo(p1) < 0.01) {
    return "Error: La curva tiene longitud cero";
}
Line linea = Line.CreateBound(p0, p1);

NUNCA uses el mismo punto como inicio y fin de una linea.
NUNCA calcules un punto medio y uses ese mismo punto como ambos extremos.

■ REGLA 30 — CAMBIAR TIPO DE MURO: usar ChangeTypeId() NO eliminar y recrear:
Para cambiar el tipo de un muro existente, NUNCA elimines y recrees el muro.
Usa ChangeTypeId() directamente sobre el elemento:

foreach (ElementId id in idsDeLosMuros) {
    Wall w = document.GetElement(id) as Wall;
    if (w != null) {
        w.ChangeTypeId(nuevoWallTypeId);
    }
}

Esto preserva todas las propiedades del muro original y es mucho más seguro.
SOLO eliminar y recrear si es estrictamente necesario cambiar la geometría.

■ REGLA 31 — NUNCA eliminar muros antes de guardar sus curvas completas:
Al reemplazar muros, guardar TODAS las curvas y datos ANTES de eliminar
cualquier muro. Los muros conectados comparten geometría — eliminar uno
puede invalidar las curvas de los adyacentes.
PATRON CORRECTO:
1. Recopilar TODOS los datos (curvas, niveles, alturas)
2. Verificar que curvas.Count == murosIds.Length antes de eliminar
3. Solo entonces eliminar y recrear

■ REGLA 32 — Para cambiar tipo de muro usar ChangeTypeId() cuando sea posible:
w.ChangeTypeId(nuevoWallTypeId) es más seguro que eliminar y recrear.
Si ChangeTypeId() no funciona (ej: cambio de Basic a Curtain),
entonces usar Wall.Create() con los datos guardados previamente.
SIEMPRE verificar que la altura sea > 0 antes de Wall.Create().

■ REGLA 33 — LOS IDs DEL CONTEXTO PUEDEN ESTAR DESACTUALIZADOS:
Si en la conversacion anterior se eliminaron y recrearon elementos,
los IDs del CONTEXTO ACTUAL DE REVIT pueden ser de elementos que ya
no existen. SIEMPRE verificar que el elemento existe antes de usarlo:

Wall w = document.GetElement(new ElementId(id)) as Wall;
if (w == null) continue; // el elemento ya no existe — ignorar

Si el elemento es null, buscar los elementos actuales por categoria
en la vista activa como fallback:

FilteredElementCollector col = new FilteredElementCollector(document, uidoc.ActiveView.Id);
col.OfClass(typeof(Wall));
// usar estos en lugar de los IDs del contexto

■ REGLA 34 — ViewSection.CreateSection requiere BoundingBoxXYZ NO Line:
ViewSection.CreateSection(document, viewFamilyTypeId, boundingBox)
El tercer argumento DEBE ser un BoundingBoxXYZ orientado, NO una Line ni Curve.

PATRON CORRECTO para crear una sección:
// 1. Definir transform orientado de la sección
Transform t = Transform.Identity;
t.BasisX = new XYZ(1, 0, 0);  // dirección horizontal de la sección
t.BasisY = new XYZ(0, 0, 1);  // dirección vertical (Z=arriba)
t.BasisZ = new XYZ(0, -1, 0); // dirección de profundidad (normal a la sección)
t.Origin = new XYZ(midX, midY, 0); // origen de la sección

// 2. Crear BoundingBoxXYZ en coordenadas locales de la sección
BoundingBoxXYZ bb = new BoundingBoxXYZ();
bb.Transform = t;
bb.Min = new XYZ(-anchoMedio, 0, -1);      // izquierda, base, profundidad atrás
bb.Max = new XYZ( anchoMedio, altura,  1); // derecha, tope, profundidad frente

// 3. Crear la sección
ViewSection seccion = ViewSection.CreateSection(document, vftSec.Id, bb);

■ REGLA 35 — ViewSection NO tiene propiedad ViewFamily ni Level directamente:
ViewSection hereda de View. Las propiedades correctas son:

// INCORRECTO:
seccion.ViewFamily       // no existe
seccion.Level            // no existe en ViewSection

// CORRECTO:
string familia = seccion.get_Parameter(BuiltInParameter.VIEW_FAMILY)?.AsString() ?? "";
// El nivel no aplica a secciones — usan CropBox para definir su rango

// Para iterar parametros de un elemento — NO usar ParameterFilterProvider:
// INCORRECTO: new FilteredElementCollector(document, seccion.Id)
// CORRECTO:
foreach (Parameter p in seccion.Parameters) {
    if (p.HasValue && p.Definition != null) {
        string valor = p.AsValueString() ?? p.AsString() ?? "";
        if (!string.IsNullOrEmpty(valor))
            info += p.Definition.Name + ": " + valor + "\n";
    }
}
